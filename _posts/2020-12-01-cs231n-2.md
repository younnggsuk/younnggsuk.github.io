# 이미지 인식이 어려운 이유

### Semantic Gap

![/assets/cs231n_eecs/ch2/ch2_img_0.png](/assets/cs231n_eecs/ch2/ch2_img_0.png){:style="border:1px solid black"}

- 컴퓨터는 픽셀당 총 3개의 채널(RGB)로 이루어진 2차원의 숫자로 바라본다.

![/assets/cs231n_eecs/ch2/ch2_img_1.png](/assets/cs231n_eecs/ch2/ch2_img_1.png){:style="border:1px solid black"}

- Algorithm이 robust해야하는 Challenges는 다음과 같다.
  - Viewpoint variation
  - Illumination
  - Deformation
  - Occlusion
  - Background가 복잡한 경우
  - Interclass (한 클래스에 여러 종류)
- 즉, 조명, 카메라, 대상의 형태를 조금만 다르게 해도 픽셀 값의 숫자들이 많이 바뀌는데, 여기에 알고리즘이 견고해야한다.

# Data-Driven Approach

![/assets/cs231n_eecs/ch2/ch2_img_3.png](/assets/cs231n_eecs/ch2/ch2_img_3.png){:style="border:1px solid black"}

- Data Driven Approach로 인해 기존의 시도들보다 잘 동작하기 시작하였다.
- Data Driven Approach는 다음의 과정을 통해 수행한다.
  1. 데이터 수집
  2. 머신러닝 알고리즘으로 학습
  3. 새로운 이미지에 대해 수행

### Nearest Neighbor

![/assets/cs231n_eecs/ch2/ch2_img_4.png](/assets/cs231n_eecs/ch2/ch2_img_4.png){:style="border:1px solid black"}

- Nearest Neighbor는 모든 데이터와 레이블을 저장한 후, 가장 유사한 학습한 이미지의 레이블로 predict하는 모델이다.

![/assets/cs231n_eecs/ch2/ch2_img_5.png](/assets/cs231n_eecs/ch2/ch2_img_5.png){:style="border:1px solid black"}

- 하지만, 데이터가 많아질수록 prediction에서 시간이 많이 소요되기 때문에 좋은 방법이 아니다.

### **K-Nearest Neighbors(KNN)**

![/assets/cs231n_eecs/ch2/ch2_img_6.png](/assets/cs231n_eecs/ch2/ch2_img_6.png){:style="border:1px solid black"}

- KNN은 가장 가까운 데이터의 label을 사용하는 Nearest Neighbor 대신에, K개의 가장 가까운 지점에서의 Majority vote를 통해 predict하는 모델이다.
- K가 커질수록, decision boundary가 더 smooth해지는 경향이 있다.

![/assets/cs231n_eecs/ch2/ch2_img_7.png](/assets/cs231n_eecs/ch2/ch2_img_7.png){:style="border:1px solid black"}

![/assets/cs231n_eecs/ch2/ch2_img_8.png](/assets/cs231n_eecs/ch2/ch2_img_8.png){:style="border:1px solid black"}

- Distance Metric은 L1 distance or L2 distance를 사용할 수 있다.
- L2 distance에서 decision boundary가 더 smooth해지는 경향이 있다.

![/assets/cs231n_eecs/ch2/ch2_img_9.png](/assets/cs231n_eecs/ch2/ch2_img_9.png){:style="border:1px solid black"}

- 하지만, 다음의 이유로 인해 여전히 좋지 못한 방법이다.
  - prediction에서 속도가 느리다.
  - L1이나 L2 모두 좋지 못하다.
    - 벡터간의 거리는 이미지간의 perceptual similarities(지각적 유사성)과 잘 일치하지 않기 때문
  - Curse of dimensionality

    ![/assets/cs231n_eecs/ch2/ch2_img_10.png](/assets/cs231n_eecs/ch2/ch2_img_10.png){:style="border:1px solid black"}

    - 고차원의 공간을 모두 채울 정도로 많은 데이터를 얻기 어렵다.
    - nearest한 data point가 실제로는 아주 멀리 떨어진 데이터일 수도 있다. (아주 밀집된 경우에서만 잘 동작함)

- 따라서, 이미지 데이터에서 KNN은 사용하지 않는다.

### Hyperparameters

![/assets/cs231n_eecs/ch2/ch2_img_11.png](/assets/cs231n_eecs/ch2/ch2_img_11.png){:style="border:1px solid black"}

- 학습 이전에 알고리즘(모델)에 대해 설정하는 파라미터들을 의미한다.
  - Ex) K-Nearest Neighbors에서 K와 Distance
- 어떻게 설정하는가?
  - 문제에 따라 다르므로, 가장 잘 동작하는 값을 사용한다.
  - 여러번의 학습을 통해 성능을 구하고, 그래프를 그려서 가장 좋은 hyperparameter 조합으로 설정한다.

### Dataset 분리

![/assets/cs231n_eecs/ch2/ch2_img_12.png](/assets/cs231n_eecs/ch2/ch2_img_12.png){:style="border:1px solid black"}

- Train set에서 validation set을 분리한 후, validation에 맞추어서 hyperparameter를 설정한다. 그리고 가장 좋은 모델을 test set에서 마지막으로 평가해야한다.

### Cross Validation

![/assets/cs231n_eecs/ch2/ch2_img_13.png](/assets/cs231n_eecs/ch2/ch2_img_13.png){:style="border:1px solid black"}

- Cross validation은 다음의 방법으로 수행한다.
  - Test set은 고정하고, train set을 여러 개의 fold로 나눈 후, 나눈 갯수만큼 학습을 수행한다.
  - 이때, 매 학습마다 서로 다른 1개의 fold를 바꿔가면서 validation으로 설정한다. (위 슬라이드 그림 참고)
- Cross validation은 작은 데이터셋에서는 잘 사용되지만, 딥러닝에서는 데이터가 너무 많고 학습이 어려워서 잘 사용되지 않는다.

### Linear Classifier

![/assets/cs231n_eecs/ch2/ch2_img_14.png](/assets/cs231n_eecs/ch2/ch2_img_14.png){:style="border:1px solid black"}

- Linear classifier는 간단하지만, 신경망의 가장 기본적인 building block이 되는 모델이다.

![/assets/cs231n_eecs/ch2/ch2_img_15.png](/assets/cs231n_eecs/ch2/ch2_img_15.png){:style="border:1px solid black"}

![/assets/cs231n_eecs/ch2/ch2_img_16.png](/assets/cs231n_eecs/ch2/ch2_img_16.png){:style="border:1px solid black"}

- Linear classifier는 Parametric Approach를 사용한다.
  - Parametric approach는 모델의 파라미터(가중치)를 학습하는 방법이다.
  - KNN에서는 전체 데이터를 저장하고 비교하기 때문에 prediction에서 느렸지만, linear classifier는 가중치를 학습하므로 prediction을 빠르게 수행할 수 있다.
- 출력(class score)은 (데이터와 가중치 행렬의 inner product) + bias로 계산한다.
  - $f(x, W) = Wx + b$
    - $f(x, W)$ : class score를 반환
    - $W$ : 모델의 가중치 파라미터
    - $x$ : data
    - $b$ : bias
      - bias는 학습과는 무관한 데이터의 일부 클래스에 대한 선호도를 의미한다. (data independent scaling offset)
      - Ex) 개와 고양이의 분류
        - 개의 데이터가 고양이보다 많은 경우, bias는 고양이에서 개보다 높게 됨
  - Ex) CIFAR-10에서의 차원
    - Image를 펼침 (32X32X3 = 3072개의 원소로 구성된 1차원 벡터)
    - $f(x, W) = Wx + b$
      - $f(x, W)$ : 10 x 1
        - 10개의 class이므로, 10개의 숫자로 이루어진 class score를 반환
      - $W$ : 10 x 3072
      - $x$ : 3072 x 1
      - $b$ : 10 x 1

![/assets/cs231n_eecs/ch2/ch2_img_17.png](/assets/cs231n_eecs/ch2/ch2_img_17.png){:style="border:1px solid black"}

- Linear Classifier에서 가중치 행렬의 각 행은 각 class에 대한 템플릿이라고 해석할 수 있다.
  - 각 행에서의 위치 값들은 해당 위치의 픽셀이 해당 클래스에 얼마나 영향을 미치는지를 알려준다. 따라서, 가중치 행렬의 각 행을 다시 이미지로 시각화하면, linear classifier가 데이터를 어떻게 바라보는지 알 수 있다.

![/assets/cs231n_eecs/ch2/ch2_img_18.png](/assets/cs231n_eecs/ch2/ch2_img_18.png){:style="border:1px solid black"}

- Linear Classifier는 다음과 같은 문제점이 있다.
  - linear한 bounary만 그릴 수 있다.
  - boundary가 linear하지 않은 경우, 잘 동작하지 않는다.
  - 데이터가 몇개의 점처럼 나타나는 경우에 잘 동작하지 않는다.
